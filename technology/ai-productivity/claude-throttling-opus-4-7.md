# Claude「降智」其實是算力危機:Opus 4.7 試玩與升級注意

> 過去兩週很多 Claude Code 重度使用者都覺得「Claude 變笨了」。這支影片拆解:**所謂降智不是模型變笨,
> 而是 Anthropic 在算力危機下偷偷下修了思考深度與配額**;接著實測剛發布的 **Opus 4.7**,整理它的新設定、
> 底層升級、benchmark,以及升級前一定要知道的三個坑。
>
> 整理自 Gary Chen(@garytalksstuff)影片逐字稿。

---

## 一、「降智」到底發生了什麼

使用體感:兩週前的 Claude 像個「會通靈的好夥伴」——你講半句、給個大方向,它就主動翻檔案、串邏輯、補細節、
還會點出你的盲點。但這兩週那種默契消失了:挑簡單的事做、忽略關鍵 context、甚至不讀檔就亂改,
偶爾還回覆看不懂的語言(例如韓文)。社群(Twitter / Reddit / 開發者群組)整個四月都在問同一句:**「今天 Claude 又降智了嗎?」**

一位 AMD 工程師拿 **近 7,000 次** Claude Code 編程對話做系統化分析,數字很誇張:

| 指標 | 變化 |
|---|---|
| 推理字元(思考深度) | 平均 **2,200 → 600**,砍掉約 **73%** |
| 不讀檔直接動手改的比例 | **6% → 33.7%**(每三次就有一次沒看 code 就改) |
| 使用者需打斷模型防止它搞砸的頻率 | 多了 **12 倍** |
| 輸出裡 "simplest" 出現頻率 | 增加近 **3 倍**(模型在偷懶,找最簡解而非對的解) |

結果就是不少訂 Max($200/月)的人,**一小時內 token 莫名燒光**。

### 為什麼會這樣:算力不夠

官方沒正面回應,但產業新聞 + Anthropic 一連串動作都指向 **算力(GPU)不足**:

- **營收爆衝、硬體沒跟上**:Anthropic 年化營收(ARR)這個月剛超過 OpenAI($300 億 vs $240 億),
  一年前還倒過來(OpenAI 是它 6 倍);15 個月衝了 30 倍,光從 90 億到 300 億只花 4 個月。
  年付超 $100 萬的大戶從二月到現在翻倍、破 1,000 家。但訓練支出據《華爾街日報》預估只有 OpenAI 的 1/4 規模,
  新 TPU 大產能要今年下半年甚至明年才陸續上線。
- **可觀察到的症狀**:三月底起對美東上午尖峰時段限流(官方承認 GPU 容量不足);4/4 企業方案從吃到飽改 pay-as-you-go;
  讓你用訂閱額度跑第三方工具(如 OpenClaw)的作法被踢掉;《Fortune》4/14 專文講效能下滑與算力危機引發的反彈。

> **本質:** 模型沒變笨,是 Anthropic「拆東牆補西牆」——預設思考深度被下修、配額拉緊、輸出 token 壓短。
> 「模型還是那個模型,只是被戴上了手銬腳鐐。」

## 二、Opus 4.7 帶來什麼

### 三個能直接感受到的新設定

1. **`xhigh`**:新增的思考努力級別,官方定位「介於 high 與 max 之間」。(作者吐槽:本來就有 max 了,
   推一個比 max 低的等級,不確定要解決什麼問題——推測見下。)
2. **`ultrareview` 指令**:Claude Code 內新增的專屬 code review,丟一個 PR 給它,用獨立視角抓 bug、找設計問題。
   對沒有隊友 review 的 solo builder 很補。**注意:它的 token 不算進 Pro/Max 額度,走額外計費;Pro/Max 各有 3 次一次性免費,用完不重置。**
3. **Max 開放 Auto mode**:讓 Claude 自己決定任務該用哪個 effort 級別,不用手動切。另外 API 推出 **Task budgets**(public beta)可設 token 預算上限。

### 底層能力升級

- **視覺解析度翻三倍**:可接受長邊 2,576px(近 375 萬像素),截圖小字、系統架構圖、Figma 設計稿都看得清。
  XBOW(資安公司)CEO 給的視覺敏銳度測試從 4.6 的 **54.5% 跳到 98.5%**(幾乎不及格→滿分)。
- **檔案系統式的跨 session 記憶更強**:把上一次對話的重點寫進檔案、下次讀回來,被列為四大亮點之一,
  做長時間多階段專案時自己帶記憶走,省下重鋪背景的工。

### Benchmark

| 測試 | 結果 |
|---|---|
| 官方 93 題 coding | 解題率比 4.6 **+13%**,還解掉 4 個 4.6/Sonnet 4.6 都解不出的難題 |
| CursorBench(IDE 場景) | **58% → 70%**,Cursor CEO 親自背書「顯著躍升」 |
| Rakuten 企業級 SWE | 能解的生產環境任務數是 4.6 的 **3 倍**,程式/測試品質雙位數成長 |
| GDPval(真實任務綜合) | **1753 分**,擊敗 GPT-5.4(1674)、遠超 Opus 4.6(1619) |
| Document & reasoning | **57.1 → 80.6**(Gemini 3.1 僅 42、GPT-5.4 僅 51,壓倒性領先) |

Hex 技術長總結最狠:**「low-effort 的 Opus 4.7 大致等於 medium-effort 的 Opus 4.6」**——你讓它偷懶,做出來還比舊版認真做的好。

## 三、升級前一定要知道的三件事

1. **token 吃得更兇**:換了新 tokenizer,一樣的輸入會被 parse 成 **1～1.35 倍** 的 token 數(最多多三成五)。
   單價沒漲,但用量拉高,帳單還是會變貴。
2. **指令跟隨變「字面化」**(最多人會踩):官方白紙黑字寫 **"4.7 takes instructions literally"**。
   你說「改這個變數」,它就只改那個變數,不再寬鬆腦補你的意圖。覺得「沒那麼機靈」不是變笨,是它更認真看待你的每個字
   → **回到好好寫提示詞的基本功**。
3. **定價沒變**:API 仍是輸入 $5 / 輸出 $25(每百萬 token)。沒漲價值得讚,但因 tokenizer + 高 effort,帳單數字高機率還是增加。

## 四、作者實測心得 & 兩個推測

- **實測有點無感**:本來預期一開 `xhigh` 就找回熟悉的 Claude,結果跑幾個常用工作流,**`xhigh` 的體驗跟 4.6 設 max 差不多**——
  沒變差,但也沒有「一拿到手就知道不一樣」的感覺。Benchmark 進步幅度很大,但日常一小時工作流體感差不多。
  > 提醒:benchmark 測的是「上限能力」(像考試滿分);日常體感測的是「平均表現」——跑一百次你的任務有幾次不出錯、思考多周全。
- **推測 1:為何有 max 還要 xhigh?** 因為這兩週就算開 max 也救不回降智。猜 `xhigh` 不是「更強」,
  而是 Anthropic 針對算力限制重新拿捏的「甜蜜點」:比 high 多一點、又不像 max 那麼燒的中間選項。
- **推測 2:更聽話的雙面性。** 更遵從指令 = 更少幻覺、更不自作主張;但反面是少了「會補位、給驚喜」的機靈,你要什麼它就剛好給什麼。

## 應用案例

- **覺得 Claude 突然變笨時:** 先別急著退訂或換模型。理解這是「算力分配的結果、短期不會消失」,
  把力氣放回**寫清楚提示詞、整理工作流、把 context 放對位置**——這些是跨模型版本都有效的基本功。
- **要不要升 Opus 4.7:** 若你常餵截圖/架構圖/設計稿,視覺三倍升級很有感;但要先意識到新 tokenizer 會讓帳單上升,
  且「字面化」指令需要你把需求講得更精確。
- **solo builder:** 試 `ultrareview` 幫沒人 review 的 PR 把關,但記得它另外計費、免費只有 3 次。
- **怕亂燒錢:** 用 Max 的 Auto mode + API 的 Task budgets,設預算上限,不用一直盯儀表板。
