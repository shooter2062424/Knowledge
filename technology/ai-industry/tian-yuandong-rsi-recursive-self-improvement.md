# 再訪田淵棟:46.5 億美金估值的 RSI,押注「AI 自進化」與前沿實驗室的組織架構之爭

> 整理自硅谷101〈再訪田淵棟:46.5 億美金估值的 RSI,與 AI 自進化｜Neolabs 特輯〉(主持人陳茜,92 分鐘長播客,有官方中文字幕)。受訪者 **田淵棟(Tian Yuandong)**——前 Meta 基礎 AI 研究(FAIR)研究總監,在 Meta 待近 11 年、去年被裁後,與另外 7 位頂級研究員共同創立新公司 **RSI(Recursive Superintelligence,遞迴超級智能)**,融資 6.5 億美元、估值 46.5 億美元。本筆記拆解:RSI 在賭什麼、為何資本買單、「遞迴自我改進」怎麼運作、前沿模型/數據/評估的技術判斷,以及他對 neolab 競爭、Meta 蒸餾爭議、AI 時代職涯的看法。

---

## 一句話總結

**RSI 賭的是「用 AI 來改進 AI 自己(遞迴自我改進,recursive self-improvement)」,把自動化 AI 科研當第一步**——因為「下一個 coding 級別的大方向就是 AI research」。田淵棟反覆強調幾個判斷:**預訓練的上限決定 RL 的上限**、語言比視覺更有信號、可解釋性在自進化路線上「比原來更重要」、而**前沿大模型之爭最後其實是組織架構之爭**(先進生產力 vs 落後生產關係的矛盾)。對個人:大廠不再需要那麼多螺絲釘,**你得進化成「四維生物」才能活下來**。

> 延伸脈絡:本片正是 [[rsi-recursive-self-improvement-anthropic]](Anthropic 為何呼籲暫停 RSI)講的那條賽道的「創業者視角」;與 [[self-harness]](讓 harness 自己改進自己)、[[karpathy-autoresearch]](Karpathy 自動化研究)同源。

## RSI 是什麼:8 個聯創、$46.5 億估值、還沒產品

- **遞迴自我改進**:用 AI 去優化 AI 自己的東西、讓 AI 變更強,「左腳踩右腳再踩上去」。終極願景是一個系統——**投進各種算力,輸出新知識、新洞察**(官網標語 *maximizing knowledge discovery rate*)。
- **為何資本在「沒有產品」時就給 46.5 億估值**:「最頂級的資本還是看人。」AI 變化太快,一開始講不清最終產品/商業願景,所以資本看的是**團隊履歷、能力、磨合程度與執行力**。
- **8 個聯創互補**:Richard Socher(CEO,前 MetaMind→Salesforce 首席科學家→You.com)、熊蔡明、**Tim Rocktäschel**(前 Meta→DeepMind,Project Genie 1/2/3 與 agent Open-Endedness 負責人,Google 口碑好、助攻融資)、**Jeff Clune**(agent Open-Endedness,《The AI Scientist-v2》被 Nature 收錄)、Josh Tobin(前 OpenAI agent science lead)、Alexey Dosovitskiy(**Vision Transformer 一作**)、Tim Shi(前 Cresta CTO)等。田淵棟負責建模/後訓練 + agentic harness 工程。
- **為何選小團隊**:大公司組織臃腫,CEO 很急、指令多,**對齊速度遠慢於小團隊**,跟不上 AI 發展速度——所以大廠頻繁 reorg、裁員、改方向。「很小很精的團隊把一件事做成」是當前趨勢。

## 核心技術判斷

### 為何「自動化 AI 科研」是第一步
- 人有生理極限(要吃喝拉撒睡),沒法 100% 時間投入;AI 可以幾千個同時跑、知識廣博、不知疲倦。
- **第一步**:讓 AI 代替繁瑣重複的研究勞動;**第二步**:讓 AI 在勞動之上**自動發現新洞察**,加速整個人類發展進程。
- **為何不選更快賺錢的方向(語音陪伴/聊天助手/垂直模型)**:那些「特化模型」過去三年已被反覆證明**有一天會被更通用的大模型蓋掉**;而「自我進化」這個**範式只要不變,公司立場就不變**,再在範式下找商業化機會。

### 評估難題與可解釋性(最重要)
- **「評估者也要被評估」的遞迴難題**:目前 AI 還達不到人的評估標準,**一段時間內人類仍可當評估者**;且 AI 越強,很多評估會**變主觀**(如代碼怎麼設計、模組關係怎麼安排,每個人見解不同、沒有唯一正解),所以評估會**從「中間 benchmark」往「最後結果」走**,變得非常 result-driven。
- **可解釋性比原來更重要**,兩個理由:① **安全**——要做 *safe* superintelligence,得理解模型內部機制才知道它怎麼運轉、將來有什麼威脅;② **效率**——遞迴自我改進要燒幾千上萬張卡,若不理解模型只能等訓練完才評估(很慢);**有可解釋性就能把評估提前**,大幅提高運算效率。RSI 內部有人專研可解釋性。

### Coconut / 潛在推理:語言不是思考本體
- 田淵棟在 Meta 的重要工作 **Coconut(連續思維鏈)**:讓 AI 在**連續潛在空間**裡推理(latent reasoning)。他認為**人思考時並不靠語言**——更多是圖像、說不清的思路,**語言是事後的「解釋」而非思考過程本身**。
- **潛在推理效率更高**:每個 latent token 能存的信息高於語言 token,**可同時存幾個不同思路並行**,最後挑對的路徑出來;用語言則得一個一個人工寫下來。Looped Transformer 也是這條 latent-space 路線(外界猜 Anthropic 的 Mythos 用了它,因其在圖搜索問題上特別強)。RSI 會繼續做,但細節保密。

### 頓悟(grokking)與泛化
- 模型**從「記憶」突然變成「泛化」**的過程,可能是解開整個神經網路訓練秘密的所在。可用**能量函數**描述:數據不足時最小值是記憶解;數據多到一定程度,能量函數結構**突變成泛化解**。理論上若總數據是 M² 量級,約需 **M log M** 的數據就能觸發(比線性多一點、但遠少於全量)。理解它就能更有效率地訓練。

### 預訓練 vs 強化學習(關鍵觀點)
- **「預訓練的上限決定 RL 的上限」**:RL 後訓練的功用是把原本排第 200 的答案提到第一——前提是**一開始至少有一個對的「種子」**;若預訓練沒把推理模式學好、連種子都沒有,**RL 再做也沒用**。所以大模型最關鍵的還是**預訓練**(各種表示在預訓練後湧現,RL 只是放大它)。RSI 會自己做預訓練(loss curve 平滑、穩定)。
- **數據往哪走**:互聯網數據已被用盡;近期一波進展靠**把推理模型的數據放回預訓練/中訓練**(初步的自我進化)。下一步是**continual learning(持續學習)**——新數據放進來學得深、又不必花那麼多代價。但要達到人類效率,最終還是得靠**新算法/新範式**,光堆數據與 scaling law 撐不住。

## Neolab 版圖與前沿實驗室大亂鬥

| 實驗室/人物 | 田淵棟的觀察 |
|---|---|
| **Ilya / SSI** | 極度保密,不知在做什麼;可能在研究**新訓練範式**(融了很多錢但未必夠做全新大模型);去年訪談關注 research taste、泛化 vs 記憶,與本系列訪談接近 |
| **Yann LeCun / AMI Labs** | 走完全不同方向(視覺**世界模型**+機器人);田淵棟認為**語言比視覺更有信號**(像素組合固定,語言可自指、無限嵌套、定義新概念),JEPA 路線「可能還太早」(對照 [[jepa-lecun-world-models]]) |
| **David Silver / Ineffable Intelligence** | 想**完全不用大模型**、純 RL 重現 AlphaZero「左腳踩右腳」的故事;但圍棋/象棋評價函數定義清楚才做得到,**開放式領域很難**、需巨量算力 |
| **Thinking Machines** | 發了持續交互的 interactive model(可一直聽你說、不用一問一答),側重人機交互 |

- **小廠的護城河 = research taste,不是 scaling law**:大廠資源近乎無限、可靠 scaling law 把事做成;小廠**得用巧妙方法繞開算力堆疊**,靠對問題更深的理解(research taste)勝出。
- **小廠 vs 大廠是「平行發展」**:大廠自上而下、追熱點導致**同質化**(coding agent 火了大家都做);小廠為快速出結果,自然選大廠現在不做的方向。歷史會重演——Anthropic 當年悶頭做 coding(不刷數學榜),等大家發現能賺錢才一窩蜂湧入。
- **時機(timing)**:**去年底 coding agent 跨過突變點**——從「AI 輔助寫代碼(Cursor)」變成「人下高層命令、AI 把整件事做掉」。整個軟體業要花 1–2 年適應,**這正是小公司彎道超車的窗口**;再晚一年大廠適應完就晚了。

### Coding 格局與下一步
- Codex 5.5 / Claude Code / DeepSeek V4 各有優劣,**沒誰比誰強很多**(Claude Code 會幻覺、認錯很快但事前不講;DeepSeek V4 要先預熱 ~1 萬 token 才變聰明)。對照 [[deepseek-v4-engineering]]。
- **隱憂**:用戶只要「夠用」的模型,超過上限就無所謂 → 最後可能**打價格戰**(除非數據飛輪成立,但「重要代碼還是大神寫的」,飛輪轉不轉得起來不確定)。
- **coding 之後的 the next big thing = AI research**:寫代碼的 ROI 高才讓 coding 爆紅;大模型人才比寫代碼的更貴,**自動化 AI research 的 ROI 更大**(發現新藥/新材料/新訓練方法都是天文數字回報)。Karpathy 加入 Anthropic 做預訓練/auto research,正說明**自我改進已成主流共識**(對照 [[karpathy-autoresearch]]、[[karpathy-software-3-0]])。

### 前沿模型之爭 = 組織架構之爭
- **xAI 的問題像 Llama 4**:老闆要求快、壓力大,組織內**沒人能 push back**,只能順從;時間久了交付與承諾不符 → 大地震 → 老闆不信任、外面再拉人,原團隊境遇慘。
- **本質**:這是**先進生產力 vs 落後生產關係的矛盾**;「大模型沒有永遠的贏家」,差異點在「**你做得多細、多努力**」——小東西有人注意到、有人沒注意到,就是差距(認同姚順宇:工業級大模型是每個人在崗位上兢兢業業把 pipeline 跑通)。

## Meta 蒸餾爭議與 AI 時代職涯

- **Meta 爭議**:裁 7800 人,同時宣布**用員工的工作數據訓練內部 AI(「蒸餾員工」)**——說得「比較赤裸裸」。田淵棟:這對員工感受不太顧及,但**會是大趨勢**(一個人做大家都會做);不過**蒸餾到底多有用是問題**——員工知道自己會被「吸星大法、頃刻煉化、然後踢出去」,可能**不願好好工作、甚至埋雷投毒**。
- **AI agent 已能做 L4/L5 工程師的活**(大廠裡 L4/L5 最多),裁員會越來越多。
- **「四維生物」職涯觀**:跳到大廠被裁、再跳再被裁,**像一條在水坑裡跳的魚,而水越來越少**(《三體》:有人把水弄乾了)。「你最終一定要變成一個四維生物才能活下來」——魚要進化成鳥、長出翅膀。
- **新文藝復興**:以前讀 4 年本科+2 年碩士+5 年博士,目的是成為大機器的螺絲釘;現在**大機器不需要你的零件了,你得自己找意義**。田淵棟 GPT-4 出來時就寫過:做一件**只有你願意去探索、別人做不了**的事,把探索得到的新數據分享出去,你就成了**「行走的數據源」**、對世界有價值。

## 應用案例 / 可帶走的判斷

- **看 AI 創業/投資**:在變化極快的領域,**先看人與執行力**,別執著於一開始就講清產品;範式級的賭注(自我進化)比特化產品更抗「被通用模型蓋掉」。
- **看模型技術**:評估一個團隊別只看 scaling law,要看 **research taste**;判斷一條路線時記住「**預訓練上限決定 RL 上限**」「**語言比視覺更有信號**」這兩個田淵棟的判準(他也說可能錯,但提供了思考錨點)。
- **看前沿競爭**:模型差距正在收斂、coding 可能走向價格戰;真正的勝負手越來越是**產品(誰讓用戶願意用)+ 組織架構(誰能快速對齊、敢 push back)**,而非單純算力。
- **個人定位**:AI 抹平 L4/L5 級別工作後,與其在大廠間跳來跳去(水越來越少),不如**找一件只有你能做、你願意深挖的事**,把產出與數據分享出去——這與本庫 [[defining-tasks-not-prompts]](人從操作者變定義者)、[[model-agnostic-ai-workflow]](把模型當可替換零件)同一個底層轉變。
