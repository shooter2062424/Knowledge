## 1. Warp — 從 AI 終端機進化成「智能代理開發環境」,正式開源

- **連結:** <https://github.com/warpdotdev/warp>
- **用途:** Warp 一開始是終端機工具,如今已是一個 **智能代理開發環境(agentic development environment)**。它內建程式設計代理(agent),也能接入你自己的 CLI 代理,例如 Claude Code、Codex、Gemini CLI 等。本週終於 **正式開源**,熱度爆表,star 數已達 "40K+"。
- **亮點:**
  - 早在 2021 年 7 月就以 "Fast, Rust-based terminal" 口號發布,設計前衛,且第一時間引入 AI 功能。
  - 使用 AI 功能需登入;啟動時可選擇 **terminal 模式** 或 **內建 agent 模式**,後者體驗近似直接進入 Claude Code 的狀態。因本身就是終端機工具,使用上更流暢、更有優勢。
  - **OpenAI 是專案的 founding sponsor(創始贊助者)**,既說明產品水準,也引發「未來是否會被收編」的聯想。
  - 已不只是終端機,還推出 **Oz** 雲端 Agent 編排平台,可同時跑最多 **40 個並發 Agent**,持續完善生態體系。
  - 創始人對「為何現在才開源」的說法值得玩味:**開發的瓶頸已經轉移了**——以前最值錢的是程式碼,現在 AI 寫得飛快、程式碼不值錢,真正重要的是人類做的事:**定義產品規格、驗證行為是否正確**。

> **應用案例:** 如果你是喜歡 CLI 工作模式的開發者,可以把 Warp 當成日常主力終端機:平時當一般 terminal 用,需要時切到 agent 模式,直接在同一個視窗裡讓內建代理改程式碼、跑指令,不必在終端機與外部 AI 工具之間來回切換;團隊任務量大時再用 Oz 雲端平台一次派出數十個 Agent 並行處理。

## 2. hackingtool — 一站式駭客滲透測試工具箱

- **連結:** <https://github.com/Z4nzu/hackingtool>
- **用途:** 網路安全領域的一把瑞士軍刀,把滲透測試各階段需要的工具整合到一個 **圖形化介面** 裡。
- **亮點:**
  - 覆蓋安全測試全流程:資訊收集(子網域列舉、連接埠掃描)、漏洞掃描、Web 攻擊(XSS、SQL 注入工具集)、無線攻擊、密碼破解、逆向工程等。
  - 基於 **Kali Linux** 執行,以 **Python** 撰寫,採 **MIT 授權**。
  - 最大優點是 **介面友善**:不用背幾十個命令列工具,點點滑鼠就能完成,對安全入門者上手成本極低。

> ⚠️ **使用聲明:** 此類滲透測試工具屬雙面刃,僅適合 **學習與合法授權範圍內的小規模測試**。對未經授權的系統進行掃描或攻擊可能觸法,請務必確認你擁有測試目標的合法授權,切勿用於不法用途。

> **應用案例:** 剛接觸資安、想動手練習的學生,可在自己架設的靶機(如 DVWA、Metasploitable)上,用 hackingtool 的圖形介面依序體驗「子網域列舉 → 連接埠掃描 → 漏洞掃描 → Web 攻擊」的完整流程,快速建立對滲透測試各階段的整體認識,而不必一開始就硬背一堆命令列工具。

## 3. Skills (mattpocock) — TypeScript 頂流的「真實工程師」Claude Code 技能集

- **連結:** <https://github.com/mattpocock/skills>
- **用途:** 作者是 TypeScript 社群頂流博主 **Matt Pocock**,他把自己日常使用的 Claude Code 技能集全部開源。上線不到 3 個月就獲得 "45000+" star,增速在開源圈相當驚人。
- **亮點:**
  - 作者特別強調:這 **不是教你 vibe coding**,而是給 **真正做工程開發的人** 用的。
  - 內含 **18+ 個技能模組**,涵蓋任務拆解、架構設計、TDD 工作流、Git 安全操作、技術文件撰寫等。
  - 安裝簡單,一行 `npx skills@latest add` 即可;每個技能 **獨立、可按需選裝**,不必全裝。安全評分拿到滿分 100 分。
  - 專案介紹寫了很多設計思路值得一讀,例如:軟體開發中最常見的故障模式是「理解偏差」,對 AI agent 也是一樣。

> **應用案例:** 一名要在既有 codebase 上加功能的工程師,可只裝「任務拆解」與「TDD 工作流」兩個技能:先讓 Claude Code 依技能把需求拆成可驗證的小步驟,再以測試先行的方式逐步實作,並用「Git 安全操作」技能避免誤刪分支或強推——把 AI 從「隨手生成程式碼」拉回到嚴謹的工程流程上。

## 4. awesome-codex-skills — 讓 Codex CLI 連接「1000+ 外部應用」的技能精選

- **連結:** <https://github.com/ComposioHQ/awesome-codex-skills>
- **用途:** 相對於上一個是 Matt Pocock 個人的技能實踐,本專案是 **社群層面的精選列表**。由 **Composio** 團隊維護,目標是讓 OpenAI 的 **Codex CLI** 能連接 "1000+" 外部應用。
- **亮點:**
  - 可做的事:自動部署、CI/CD 流程自動化、程式碼審查,以及連接 Slack / GitHub / Jira 等外部服務——把日常需手動操作的工作流交給 AI 自動跑。
  - 每個技能都是 **模組化** 的,可獨立安裝、按需組合。
  - 作者觀點:大家常聽到分享 Claude Code,卻忽略了 Codex;近期 Codex 更新與 GPT 模型能力提升都很不錯,加上對中國區相對友善、用量充足,是近期很值得使用的開發方案。
  - 直觀對比:作者讓 DeepSeek 與 Codex 同時開發一個俄羅斯方塊的 iOS 應用,DeepSeek 花了 30 多分鐘且沒做 iOS 專案初始化,Codex 則 **8 分鐘** 就產出初版,差距明顯。

> **應用案例:** 一位獨立開發者想把「PR 合併後自動部署、並在 Slack 通報結果」變成全自動流程:他從 awesome-codex-skills 挑選對應的部署、CI/CD 與 Slack 技能裝進 Codex CLI,之後只要在終端機下一句指令,Codex 就會串起 GitHub、部署平台與 Slack,把原本要手動點好幾個介面的工作一次跑完。

## 5. Pixelle-Video — 輸入一個主題,全自動生成短影片

- **連結:** <https://github.com/AIDC-AI/Pixelle-Video>
- **用途:** 一個實用的 AI 工具,賣點是 **輸入一個主題,輸出一個完整的短影片**。
- **亮點:**
  - 全自動流程:AI 寫文案 → 依文案生成配圖 → 逐幀處理成影片 → 自動合成語音解說 → 匹配背景音樂 → 最後合成成品。
  - 每個環節都可 **自訂**:換不同 AI 模型生成圖片、換不同語音引擎、調整視覺風格等,**模組化設計** 靈活性不錯。
  - 作者經驗談:除了工具本身,**模型能力** 才是關鍵,而好模型成本很高(例如 seedance 2.0 能力很強,但生成影片價格也偏貴)。專案本身不算複雜,推測可能是 vibe coding 的產物,適合當作學習與參考——有興趣者完全可以自己 vibe coding 出類似工具。

> **應用案例:** 經營知識型帳號、但不擅長剪輯的創作者,只要輸入「3 個提升睡眠品質的方法」這樣一個主題,Pixelle-Video 就能自動寫好腳本、配上插圖與旁白、加上背景音樂並合成成片;若覺得畫風不合,再切換圖片模型或視覺風格重跑,把製作一支短影片的門檻從「會剪輯」降到「會打一句話」。
