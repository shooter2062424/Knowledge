## 1. OpenClaw — 跑在自己裝置上的個人 AI 助理

- **連結:** <https://github.com/openclaw/openclaw>
- **用途:** 一款 **運行在你自己裝置上** 的個人 AI 助理,可 7×24 小時待命,並能用你熟悉的聊天工具(如 WhatsApp、Telegram、iMessage)直接操控。它在短短幾天內三度改名(clawdbot → moltbot → OpenClaw),是本期最大熱點。
- **亮點:**
  - 熱度驚人:週初約 20K star,到週末已突破 **100K star**,star 增速比作稿速度還快;甚至被說帶動了當季 Mac mini 銷量。
  - 支援 **macOS / iOS / Android**;各大雲服務商第一時間提供支援,本地部署或雲端部署(如阿里雲)皆可,上手選擇多。
  - 作者本身已是創業有成、財富自由的億萬富翁,卻仍回頭親自寫程式碼,話題性十足。
  - 目前的爭議集中在兩點:**安全性**(普遍共識認為問題仍多,待持續強化)與 **實用性**(是否真能帶來效率提升仍受質疑)。

> ⚠️ **使用聲明:** OpenClaw 能用聊天軟體操控你自己的裝置、長期待命存取個人資料,等同把高權限交給一個 AI 代理。目前社群普遍認為其安全性尚未成熟,建議在隔離環境試玩、避免直接接觸敏感帳號與資料,並自行評估風險。

> **應用案例:** 你想要一個全天候待命的「私人秘書」,人在外面用 Telegram 傳一句「幫我整理今天信箱、把重要的列出來」,它就在家裡的 Mac mini 上代為操作並回報結果;不必開電腦遠端連線,直接用手機聊天框下指令即可。

## 2. Remotion — 用 React 寫程式生成影片

- **連結:** <https://github.com/remotion-dev/remotion>
- **用途:** 一句話講就是 **用 React 程式碼生成影片**。你可以把影片當成一個 React 元件來寫,CSS / Canvas / SVG / WebGL 等 Web 技術都能直接用;同時用變數、函式、API、數學與演算法做動畫——本質上是「把剪輯時間線變成程式碼」。
- **亮點:**
  - 專案其實已存在一段時間,最近爆紅是因為 **支援了 agent skill**,網上許多人實測後反映效果不錯,越傳越火。
  - 用法:找一個支援 agent skill 的工具,叫它幫你安裝這個 skill(給它連結即可),再用自然語言描述影片內容;流程大致是建立 Remotion 專案、設計分鏡、撰寫 React 元件、實作動畫邏輯。
  - 開發完成後啟動服務,可打開 **Remotion Studio** 預覽生成內容並匯出影片。
  - 作者實測用 **GLM4.7** 與 **Kimi K2.5** 各跑一次比較效果;整體適合做一些簡單內容,未來可期,但也沒到神乎其技的程度。

> **應用案例:** 你要替每週的數據報告自動產出一支開場動畫影片,把當週數字、長條圖動畫用程式參數帶進去。傳統做法得在剪輯軟體手動改;用 Remotion,只要更新資料變數,跑一次就重新算出整支影片,還能交給 agent 自動完成大半流程。

## 3. Kimi K2.5 — Kimi 最新開源模型

- **連結:** <https://huggingface.co/moonshotai/Kimi-K2.5>
- **用途:** Kimi(月之暗面)發布的新一代開源模型,在 **代理(agent)、程式設計與視覺** 上都有亮眼表現,上線不到 24 小時就在多個權威榜單拿下好成績。
- **亮點:**
  - 官方主打 **程式能力跨越式提升**,明顯針對程式碼場景做了優化;網上不少人稱讚 K2.5 的 **前端審美突出**,能做出設計師水準的高級美感與動效頁面。
  - **視覺能力**:可上傳影片 demo 後直接據此進行開發,相當吸引人。
  - 同步推出 **Kimi Code**(自家版的 Claude Code 式工具),以及一個尚在嘗鮮中的 **agent 集群(多代理)能力**。
  - **成本優勢明顯**:目前測試顯示 K2.5 成本約為 GPT 的 **1/5**,在各種工具都很「燒 token」的當下很有競爭力。

> **應用案例:** 你要把一段產品操作影片做成可互動的前端展示頁。傳統得先人工拆解影片需求、再寫程式;用 K2.5 把 demo 影片丟給它,讓它直接生成具備高級動效的前端頁面,且因為單價只有 GPT 約 1/5,反覆迭代調整也不心疼 token 帳單。

## 4. PageIndex — 不靠向量、不切塊的新型 RAG

- **連結:** <https://github.com/VectifyAI/PageIndex>
- **用途:** 由 VectifyAI 開源的 **新型 RAG**,提出 **不依賴向量資料庫、不對文件切塊** 的檢索方法,改以 **結構化推理 + 樹狀索引**,讓 AI 對複雜、長篇文字做高準確度檢索。
- **亮點:**
  - 直指傳統向量式 RAG 的核心痛點——**相似性與相關性的矛盾**;面對金融、法律、學術等需要專業推理的長文件時尤為明顯。
  - 運作思路兩步:(1) 先從文件自動建立 **樹狀索引**;(2) 再透過 **推理 + 樹搜尋** 模擬人類專家的查閱方式檢索答案,不再仰賴向量比對。
  - 提供 **Python SDK、線上 API 與 MCP**,後續可能再推出搭配的 skill。
  - 官方宣稱在 benchmark 上達到最先進水準,但屬探索型新技術,實際效果仍需多觀察。

> **應用案例:** 法務團隊要在一份上百頁的併購合約裡找出「違約金條款與其前後關聯」。向量檢索常因切塊而把上下文打散、抓回一堆「字面相似但不相關」的片段;PageIndex 先把合約建成章節樹,再像人一樣一層層往下翻找到對應條款,命中更精準。

## 5. LangExtract — 從非結構化文字抽取結構化資料

- **連結:** <https://github.com/google/langextract>
- **用途:** Google 開源的 Python 庫,用來從 **非結構化文字**(報告、合約、郵件、臨床紀錄等)抽取 **結構化資訊**,輸出可驗證、可追溯、符合使用者自訂 schema 的資料。底層以 LLM 為引擎,透過設定 prompt 與少量範例,自動辨識關鍵資訊並組織成嚴格定義的結構;常與 RAG、agent 搭配使用。
- **亮點:**
  - 用法極簡:`pip install langextract` 後在 Python 中使用;你需要定義模型提示詞,並給一個高品質範例來引導模型。
  - 支援生成 **互動式 HTML 報告**,以高亮方式對照「抽取結果」與「原始文字」,方便人工核對與稽核。
  - 官方示範用《羅密歐與茱麗葉》解析,能抽出角色、情感與關係並做視覺化呈現。
  - 本質上是把大模型的抽取能力做了更好的 **工程化封裝**。

> **應用案例:** 醫院要把成千上萬份自由書寫的臨床紀錄,整理成「病患、診斷、用藥、劑量」的結構化表格。用 LangExtract 定義好 schema 與一兩個範例,即可批次把雜亂文字轉成可驗證的欄位資料,並用高亮 HTML 報告讓醫師快速核對抽取是否正確。
