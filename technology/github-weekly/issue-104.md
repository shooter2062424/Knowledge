## 1. GLM-5 — 智譜新一代開源旗艦模型

- **連結:** <https://github.com/zai-org/GLM-5>
- **用途:** 智譜(Z.ai)發布的新一代旗艦大語言模型,核心口號是「**從會寫程式碼走到會完成任務**」,強調從程式碼到工程的 **端到端應用開發**,因此大幅強化了編程能力與 agent 能力。
- **亮點:**
  - **744B 總參數、40B 激活參數**,使用 **28.5T** 預訓練數據。
  - 採用與 DeepSeek 同款的 **稀疏注意力機制**,整合 DSA 技術,在保持長上下文能力的同時 **大幅降低推理成本**。
  - 能力基本與 **Claude Opus 4.5 一致**,發布當下為 **開源模型第一**。
  - 官方 demo 展示了「學術版 TikTok」、可連線對戰的大富翁遊戲等應用。
  - 與其幾乎同步發布的還有 **MiniMax-M2.5**,可用同一組 Claude Code 提示詞互相對比效果。
  - ⚠️ **本地運行不友善**:LM Studio 裡最小的 GGUF 量化版也要約 **176G**,一般機器難以承載。隨著模型上市與更新加速,GLM coding plan 的訂閱價格也從約 800 多元漲到 1200 多元。

> **應用案例:** 一個需要端到端完成需求的 vibecoding 使用者,想把「把一段需求描述變成可跑的完整小應用」這件事交給模型——
> 例如要它生成一個可連線對戰的小遊戲或學術短影片牆,GLM-5 能從規劃、寫程式碼到串接組裝一次完成,而非只丟回一段片段程式碼。
> 本期影片裡作者甚至直接讓 GLM-5 自己生成這一段專案介紹文,再拿 MiniMax-M2.5 用同樣提示詞輸出做盲測對比,就是典型的「丟一個完整任務、比拚誰能真的做完」的玩法。

## 2. cc-switch — AI 編碼助手配置管理器

- **連結:** <https://github.com/farion1231/cc-switch>
- **用途:** 一個開源的 **跨平台桌面應用**,為各種 AI 編碼助手提供 **統一、可視化** 的設定與管理入口。最初是為 Claude Code 做的,後來逐步擴大支援 Codex、Gemini CLI 等常見 AI coding 工具。
- **亮點:**
  - 解決傳統「手動編輯設定檔來切換 API key、模型、MCP server」的繁瑣流程,改用 **一個介面就能切換、備份與管理** 所有相關設定。
  - 有圖形介面,對動手能力不強的非專業族群(尤其是 **非程式設計師的 vibecoding 愛好者**)非常友善。
  - macOS 可直接透過 **brew 安裝** 使用。
  - 專案頁面有不少贊助,反映它同時獲得使用者與廠商(甲方)的雙重認可——把一個實用小工具做好,就能有不錯的效果與收益。

> **應用案例:** 一位重度 vibecoding 使用者同時開了好幾家的 coding plan(因為單一模型的額度常常不夠用),
> 過去要在 Claude Code、Codex、Gemini CLI 之間切換,得手動改各自的設定檔、貼來貼去 API key。
> 改用 cc-switch 後,只要在一個視窗裡點選就能切換目前要用哪個模型/服務商,還能一鍵備份設定,省去翻找與貼錯 key 的麻煩。

## 3. shannon — 全自動 AI 滲透測試工具

- **連結:** <https://github.com/KeygraphHQ/shannon>
- **用途:** 一個開源的 **全自動 AI 滲透測試工具**,核心目標是 **自主發現並驗證 Web 應用的可利用漏洞**,填補「日常開發」與「年度滲透測試」之間的安全缺口。它不只是掃描漏洞清單,還會像人類紅隊工程師一樣分析程式碼/運行環境,並 **實際做爆破驗證漏洞是否真實存在**,等於請了一個全年無休、24 小時在線的紅隊黑客幫你挖洞。
- **亮點:**
  - 將 **白盒原始碼分析** 與 **黑盒動態利用** 結合,分為四個由不同 agent 執行的階段:
    - 原始碼感知分析(白盒檢測)
    - 自動化瀏覽器驅動的攻擊驗證
    - 多階段攻擊流程(偵查 → 漏洞分析 → 利用)
    - 並行執行與報告生成
  - 使用方式:直接 clone 專案到本地、執行專案腳本,會 **自動完成相關 Docker 下載**。
  - 需準備自己的大模型 key;目前 **不支援自訂 baseurl**,所以只能用 Claude、OpenAI,或走 OpenRouter。
  - ⚠️ 這類工具相當 **耗 token**,需有預算心理準備。

> ⚠️ **使用聲明:** 滲透測試/漏洞利用工具屬雙面刃,僅供 **對自己擁有或已取得合法授權的系統** 進行安全測試與研究。
> 未經授權對他人系統掃描、爆破或利用漏洞可能觸犯法律,請務必先確認授權範圍與當地法規。

> **應用案例:** 一個小團隊養不起每季外包滲透測試,平時只靠 CI 上的靜態掃描,但掃描器報的多半是「疑似」漏洞、誤報一堆。
> 把 shannon 接到自家測試環境後,它會白盒讀程式碼找出可疑入口,再自動開瀏覽器去實際打、嘗試爆破,
> 最後只回報「**真的被打穿**」的漏洞並附上利用路徑報告——大幅減少人工逐一驗證誤報的工作量。

## 4. monty — 安全的 Python 解釋器

- **連結:** <https://github.com/pydantic/monty>
- **用途:** 由 **Pydantic 團隊用 Rust 編寫** 的一個極簡、安全的 Python 解釋器實作,專為 **AI 系統(尤其是 agent)的嵌入式程式碼執行** 場景設計。它的目標不是取代完整的 CPython,而是提供一個 **高速、安全、受控** 的環境,讓 AI 生成的 Python 程式碼能在內部安全運行,而不必把宿主系統的檔案系統、網路或其他敏感資源暴露出去。
- **亮點:**
  - 核心特性:**安全沙箱、極致啟動速度、受控的執行子集、多語言綁定**。
  - 針對 agent 場景的常見需求設計:
    - 執行模型生成的腳本(例如分析數據、操作結構體等)
    - 在運行時動態調整行為(生成並執行邏輯片段)
    - 保證安全性(程式碼不應能任意讀寫磁碟或網路)
  - 相較於傳統做法(啟動完整 CPython 或容器沙箱,前者有安全隱患、後者啟動與通訊開銷大且複雜),Monty 更 **輕、更安全、更易嵌入**。

> **應用案例:** 一個會「自己寫程式碼來算答案」的 agent(例如使用者問「幫我把這份報表的某欄做標準差」,模型生成一段 Python 來算),
> 直接跑模型產的程式碼風險很高——萬一它寫出讀取 `~/.ssh` 或對外連線的程式碼就糟了。
> 把執行環節換成 Monty,程式碼跑在無檔案系統、無網路的受控沙箱裡,啟動又快到幾乎沒有感覺,
> 既能讓 agent「動手算」,又不必擔心它越權碰到主機資源。

## 5. dexter — 深度金融研究智能體

- **連結:** <https://github.com/virattt/dexter>
- **用途:** 一個開源的 **自治 agent(Autonomous Agent)**,專門為 **深度金融研究與自動化分析** 而建。它能把複雜的財務問題,轉換成「**結構化研究計畫 → 自動數據獲取 → 分析與驗證 → 可信結論輸出**」的完整任務流,而不是只回一段簡單答案。整個流程強調「**先思考再行動 + 自我檢驗結果可信度**」,這在傳統金融分析自動化工具裡較為罕見。
- **亮點:**
  - 能像真正的量化研究員/金融分析師那樣:
    - 理解複雜金融查詢
    - 拆解成可執行的研究步驟
    - 抓取即時市場數據與報表資訊
    - 檢驗結果的可靠性並修正
    - 輸出結構化、可複用的分析結果
  - 安裝/使用流程:可用 **一鍵腳本** 安裝;接著把專案 clone 到本地、執行 `bun install`,再把 `env.example` 複製成 `.env` 並填入大模型 key 與各種數據 API key,設定好後 `bun start` 即可。
  - ⚠️ 它使用的 **dataset 需付費且偏貴**,實際上手門檻較高。

> **應用案例:** 想回答「**比較某兩家公司過去三年的毛利率趨勢,並判斷誰的成長更具持續性**」這種需要多步驟的問題,
> 一般聊天機器人只會憑記憶給個粗略回答。Dexter 則會先擬定研究計畫,自動去抓兩家公司的財報與即時市場數據,
> 做完計算後 **再自我檢驗結論是否可靠**、必要時修正,最後輸出一份有數據佐證、可重複使用的結構化分析——比較接近一位初階研究員交出的草稿。
