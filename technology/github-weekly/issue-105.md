## 1. zeroclaw — Rust 版 OpenClaw 替代執行時

- **連結:** <https://github.com/zeroclaw-labs/zeroclaw>
- **用途:** 隨著 OpenClaw 紅遍各地,各路衍生專案遍地開花。ZeroClaw 是用 **Rust** 編寫的 OpenClaw 替代品,定位為 **高效能、低資源占用、可組合的自主代理(agent)執行時**。它把模型、工具、記憶與執行層抽象出來,讓 agent 做到「一次建構、隨處運行」,可視為面向 agent 工作流的「執行時作業系統」。
- **亮點:**
  - **效能與資源優勢:** 與許多基於 Node.js / Python 的 agent 相比,在記憶體需求、啟動速度、套件體積上都更省;與 OpenClaw、NanoBot、PicoClaw 對比皆有優勢——這正是來自 Rust 本身的特性。
  - **Trait 驅動的可插拔架構:** 透過 Rust 的 trait 介面把核心模組化,providers、channels、memory、tools 全是可插拔元件,可依需求替換。
  - **小體積、低消耗:** 更適合在小型/邊緣設備上發揮。
  - **安全預設(secure by default):** 強化安全方面的設計。
  - 形象是一隻螃蟹,讓現在的 AI 圈變成一場「螃蟹龍蝦大打出手」的海鮮大戰。

> **應用案例:** 想在樹莓派之類的小型裝置上常駐一個自主 agent,但 Node.js / Python 版本啟動慢、吃記憶體又佔空間——
> 改用 ZeroClaw,靠 Rust 的低消耗常駐運行,再透過 trait 把記憶層換成本地檔案、把 provider 換成本地模型,組出一套輕量又安全的邊緣 agent。

## 2. voicebox — 本地運行的語音克隆工具

- **連結:** <https://github.com/jamiepine/voicebox>
- **用途:** Voicebox 是 **完全在本地運行** 的語音克隆工具,被稱為「語音克隆領域的 Ollama」。它基於 **Qwen3-TTS**,只需幾秒鐘音訊就能克隆任意聲音,語音數據完全不離開設備。
- **亮點:**
  - **下載即用:** 直接到官網依自己的系統選版本下載,能快速下載語音模型,從幾秒音訊克隆任意聲音,並用編輯工具創作多聲部專案。
  - **隱私優先:** 無需訂閱、無需上傳雲端,語音數據不離開設備,兼顧效果與隱私。
  - **UI 友善:** 介面比用 Streamlit 簡單拼出的 UI 友好太多,對新手上手體驗好很多。
  - **不只克隆:** 除了用 Qwen-TTS 做聲音克隆,還支援用 **Whisper** 做翻譯,模型都能快速下載。
  - **對 Mac 友善:** **MLX 後端** 帶原生 Metal 加速,蘋果晶片推理速度快 **4–5 倍**。
  - (作者註:很多創業與開源專案背後其實都選用 Qwen 的衍生模型,並非沒人用。)

> **應用案例:** 你想替一段中文影片配上自己的旁白,但又不想把聲音上傳到任何雲端 TTS 服務——
> 用 Voicebox 錄幾秒自己的聲音建立聲線,本地生成整段旁白;在 MacBook 上靠 MLX 加速,幾分鐘就跑完,且原始語音始終留在自己機器上。
>
> ⚠️ **使用聲明:** 語音克隆屬雙面刃,僅用於克隆 **你本人或已獲明確授權** 的聲音。未經同意克隆他人聲音可能涉及偽冒、詐騙與肖像/聲音權侵害,請務必確認授權與合規。

## 3. Qwen3.5 — Qwen 最新原生多模態開源大模型

- **連結:** <https://github.com/QwenLM/Qwen3.5>
- **用途:** 就在除夕當天,Qwen 發布了開源大模型最新版本 **Qwen3.5**。它在開源領域影響力最大,是本期最受關注的模型之一。
- **亮點:**
  - **原生多模態:** 從訓練之初就讓視覺與語言在統一表徵空間中聯合學習,在 GUI 理解、影片分析等場景可「一模到底」,符合當前模型發展趨勢。
  - **創新混合架構:** 將 **線性注意力(Gated Delta Networks)** 與 **稀疏混合專家(MoE)** 結合,力求在能力、速度與成本間取得最優平衡。
  - **旗艦模型:** 除夕開源的是 **Qwen3.5-397B-A17B**(397B 總參數、激活 17B),體積大、本地很難跑。
  - **中等規模系列:** 本週又推出 **Qwen3.5-122B-A10B、Qwen3.5-35B-A3B、Qwen3.5-27B** 等;其中 **Qwen3.5-35B-A3B** 最受關注,這個大小可在許多本地與邊緣設備上運行。
  - **實測:** 作者本地試用,圖片理解效果不錯、速度非常快;不過複現「春晚會場」這類含圖像理解 + 編程的 case,與 QwenChat 線上版仍有明顯落差。

> **應用案例:** 你想搭一套本地、又能看圖又能寫程式的 agent 工作流,但旗艦 397B 跑不動——
> 選 Qwen3.5-35B-A3B 在本地起服務,搭配 OpenClaw 之類的 agent 框架做圖片處理 / 程式生成;靠 MoE 激活參數少、推理快的特性,在單機上也能維持堪用的速度與多模態能力。

## 4. pentagi — 全自動 AI 滲透測試系統

- **連結:** <https://github.com/vxcontrol/pentagi>
- **用途:** PentAGI 由 **VXControl** 團隊開發,是開源的 **全自動 AI 滲透測試與紅隊工作流系統**。目標是讓 AI 不只「生成漏洞清單」,而是像資深安全工程師那樣 **計劃、執行、驗證、報告**。你只需輸入一個目標域名,系統就自動執行偵察、連接埠掃描、漏洞挖掘,甚至生成完整報告,全程無需人工介入。
- **亮點:**
  - **整合 20+ 主流安全工具:** 包含 nmap、Metasploit、sqlmap、nikto、gobuster、hydra 等,由 AI 自主調度、動態決策攻擊流程。
  - **沙盒隔離:** 所有操作都在沙盒化的 Docker 環境中進行,完全隔離。
  - **安裝:** 需要 Docker,下載 installer 安裝,過程中配置大模型、Search engine、SSL 等;裝好後有 Web UI 可訪問。
  - **架構文件詳盡:** 公開了容器架構、實體關係、代理互動、記憶系統等具體設計,很有研究與學習價值。

> **應用案例:** 一家公司剛上線新的對外網站,安全團隊想在正式對外前先做一輪自動化滲透——
> 把測試目標域名丟給 PentAGI,讓它在隔離的 Docker 沙盒裡自動完成偵察、連接埠掃描與漏洞挖掘,最後產出一份可交付的滲透報告,省下大量手動跑工具的時間。
>
> ⚠️ **使用聲明:** 滲透測試工具僅可對 **你擁有或已取得書面授權** 的系統使用。未經授權對他人系統掃描/攻擊在多數司法管轄區皆屬違法,請務必確認測試範圍與授權後再使用。

## 5. carbon — 精美原始碼圖片生成工具

- **連結:** <https://github.com/carbon-app/carbon>
- **用途:** Carbon 能輕鬆把你的原始碼生成漂亮的圖片並分享。看到各種文章裡好看的程式碼片段?Carbon 就能幫你快速做出同樣的效果。
- **亮點:**
  - **直接用網站:** 介面簡潔,把程式碼換成自己的即可;支援 JavaScript、Python、C++ 等多種主流語言,並可切換不同顯示樣式。
  - **PWA 離線使用:** 可保存成 PWA 應用離線使用。
  - **多種整合:** 提供各種編輯器外掛與 CLI 工具,對開發者非常實用。

> **應用案例:** 你要在技術部落格或社群貼文展示一段關鍵程式碼,但直接貼純文字既不好看也容易被排版破壞——
> 把程式碼貼進 Carbon,選一套順眼的主題與字型,匯出成 PNG,就能得到一張排版精美、語法高亮的程式碼圖片直接分享。
