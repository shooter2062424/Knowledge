## 1. CoPaw — 阿里通義的 OpenClaw 替代

- **連結:** <https://github.com/agentscope-ai/CoPaw>
- **用途:** 通義實驗室推出的個人 AI 助理,**可本地也可雲端部署**。介面一看就知道對標的是近期爆紅的 OpenClaw(後者已超過 26 萬 star,是 GitHub 上 star 最多、增長最快的軟體類專案),如今大廠也下場參戰。
- **亮點:**
  - **部署超簡單**:1 條命令即可本地部署,也支援 `pip install` 安裝;裝好後 `copaw init --defaults` 做初始配置(沒想好的選項可先略過,之後在 web UI 補設),最後 `copaw app` 啟動服務,瀏覽器開 **8088 埠** 即可使用。
  - **多聊天平台串接**:可連接釘釘、飛書、QQ、Discord、iMessage 等,支援定時任務與主動心跳,並透過 **Skills 機制** 無限擴充能力——思路與 OpenClaw 基本一致。
  - **內建主流雲端模型**:支援魔搭、DashScope,最新版還加入自訂模型供應商與 ollama。內建若干技能,也可自建/匯入。
  - **核心元件解耦**:最近升級實現了 Prompt、Hooks、Tools、Memory 等核心元件解耦,開發者可獨立替換或擴充任意模組,按需組裝自己的 Agent。
  - 整體已重現 OpenClaw 的框架,但實作仍稍嫌粗糙、待打磨;優點是 web UI 比 OpenClaw 好用、更貼合國內習慣,後續關鍵在能否維持高速迭代。

> **應用案例:** 想要一個「中文圈優先、可接釘釘/飛書」的個人助理,又不想被綁在 OpenClaw 上——
> 例如把 CoPaw 部署在自家伺服器,接上飛書機器人後設定定時任務「每天早上彙整未讀群訊息摘要」,並用 Skills 機制自行加掛「查內部 wiki」技能,全程資料留在本地。

## 2. RuView — 用 WiFi 訊號感知人體活動

- **連結:** <https://github.com/ruvnet/RuView>
- **用途:** 一個非常有意思的開源專案:**用 WiFi 訊號而非攝影機** 來實現人體感知,包括人體姿態估計、呼吸偵測、心率監測與存在偵測。目標是把普通 WiFi 設備變成「隱形感測器」,不需任何攝影機或穿戴裝置就能感知房間裡人的活動狀態。
- **亮點:**
  - **核心原理是 WiFi 的 CSI(Channel State Information,通道狀態資訊)**:人體移動或呼吸會改變 WiFi 訊號的傳播路徑與相位,RuView 透過訊號處理與機器學習分析這些變化,推斷姿態、呼吸頻率甚至心率。
  - **硬體要求**:想實際玩,務必選用 **支援 CSI 的設備**。
  - **快速上手**:可用專案提供的 docker 映像快速部署。
  - 適用情境:智慧建築(房間佔用偵測、會議室使用統計、自動控制 HVAC 與照明)、醫療監測(非接觸式呼吸與心率)、安全監控(跌倒偵測、存在偵測)、工業環境(煙霧/灰塵/遮蔽下的人員監測)。

> ⚠️ **使用聲明:** 透過 WiFi 感知他人活動涉及隱私,請僅在自有空間、取得當事人同意或合法授權的前提下使用,並遵守當地隱私與監控相關法規。

> **應用案例:** 在獨居長輩家中部署一台支援 CSI 的路由器搭配 RuView,不裝攝影機(顧及隱私)就能做「非接觸式跌倒偵測 + 呼吸異常告警」;或在會議室用它統計實際佔用率,讓中央空調與照明依人數自動開關省電。

## 3. worldmonitor — 即時全球情報監控

- **連結:** <https://github.com/koala73/worldmonitor>
- **用途:** 一個開源的 **即時全球情報監控儀表板**。它把新聞、地緣政治數據、市場資訊以及全球關鍵基礎設施數據整合到同一個視覺化介面,讓使用者像看「作戰指揮室大屏」一樣,快速掌握世界各地正在發生的事。
- **亮點:**
  - 提供多個 **live demo** 可直接體驗效果(例如可聚焦伊朗等熱點地區資訊)。
  - **本地部署**:可在 release 下載對應作業系統的版本;使用時需設定大模型的 API key,並可自行調整展示模組與資訊來源。
  - 技術門檻其實不高(與許多博主分享過的「全網資訊採集」思路相近),但非常實用,且「作戰指揮室大屏」的視覺風格相當吸睛。

> ⚠️ **使用聲明:** 此工具會抓取多方公開資訊源,使用時請遵守各資料源的服務條款與授權,並注意所引用情報的真實性與時效,避免誤判。

> **應用案例:** 地緣政治關注者或外貿/能源從業者,把 worldmonitor 部署在一台閒置主機上接成牆面大屏,即時盯著「中東衝突新聞 + 油價走勢 + 海運航線」三合一面板,在重大事件爆發的第一時間掌握對手部署與市場反應。

## 4. GitNexus — 程式碼倉庫生成知識圖譜

- **連結:** <https://github.com/abhigyanpatwari/GitNexus>
- **用途:** 一個開發者工具,可把整個程式碼倉庫自動轉換成 **知識圖譜(Knowledge Graph)**,幫助開發者用圖結構理解程式碼之間的關係。只要把一個 GitHub 倉庫或 ZIP 包拖進去,系統就會自動生成程式碼依賴關係、模組結構與互動關係,並提供可視化探索介面。
- **亮點:**
  - **內建 Graph RAG Agent**:可基於生成的知識圖譜回答問題,例如「某個函式在哪裡被呼叫」「哪些模組之間存在依賴關係」。
  - **「零伺服器」架構**:可完全在瀏覽器內運行——程式碼解析、圖譜建構、向量 embedding 與查詢分析全在客戶端完成,程式碼不會上傳到任何伺服器,對隱私友善。
  - **CLI + MCP 模式**:除了 Web UI,還能 `npm install -g gitnexus` 安裝 CLI 工具,搭配 MCP 做分析,擴充性更佳。
  - 專案有詳細介紹自身技術棧與架構,適合深入研究。

> **應用案例:** 接手一個陌生的大型 codebase 時,把整個 repo 拖進 GitNexus,先用圖譜看清「核心模組互相依賴的拓樸」,再問 Graph RAG Agent「登入流程從哪個 handler 開始、會呼叫到哪些服務」,幾分鐘就建立全局心智圖,而不必逐檔 grep——與本庫程式工具中「結構化理解程式碼」的路線一致。

## 5. deer-flow — 多代理 Deep Research 框架

- **連結:** <https://github.com/bytedance/deer-flow>
- **用途:** 字節跳動開源的 **Deep Research 多代理框架**,目標是讓 AI 不只是回答問題,而是能 **自動做研究、收集資訊、分析數據並生成完整成果**。它把大語言模型與一系列工具(Web 搜尋、網頁爬蟲、Python 執行環境等)結合,讓 AI 完成複雜的研究任務。
- **亮點:**
  - **核心能力**:Deep Research 自動化、多代理協作架構、工具與技能系統、沙箱執行環境。
  - **2.0 版從零重寫**:與 v1 不共享任何程式碼,主要基於 **Python + LangGraph + LangChain** 構建。
  - 透過 **DAG 工作流** 組織多個 Agent 的執行流程,支援長時間任務、記憶機制與任務檢查點(checkpoint)。
  - 主要應用方向:深度研究、內容創作、數據管道、工作流自動化。

> ⚠️ **使用聲明:** 框架內含網頁爬蟲與程式碼執行環境,爬取資料請遵守目標網站的 robots 與服務條款;沙箱執行外部/AI 產生的程式碼時注意隔離與安全。

> **應用案例:** 要寫一份「某新興賽道的市場研究報告」,用 deer-flow 下達主題後,框架自動拆解子問題、派多個 Agent 分頭做 Web 搜尋與網頁爬取,再用 Python 環境清洗數據、繪圖,最後彙整成帶引用的完整報告;因支援檢查點,長任務中途中斷也能續跑。
