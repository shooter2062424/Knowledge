## 1. NanoBot — 用約 4000 行復刻 OpenClaw 的超輕量個人 AI 助理

- **連結:** <https://github.com/HKUDS/nanobot>
- **用途:** 香港大學數據科學實驗室(HKUDS)開源的超輕量個人 AI 助理,靈感來自爆紅的 OpenClaw。OpenClaw 那足足 43 萬行的程式碼量勸退了很多人,而 NanoBot **僅用約 4000 行就復刻了 OpenClaw 的核心功能**,真正做到超輕量。
- **亮點:**
  - 架構參考 OpenClaw:從聊天工具接入,一路到底層的執行。
  - 三個最核心的能力:**自然語言驅動的任務執行**、**記憶模組(持久化上下文)**、**插件化的動作/工具呼叫**。
  - 接入端目前支援 **Telegram、WhatsApp、飛書** 等聊天工具。
  - 模型同時支援 **雲端 API 與本地模型**,在意成本可改用本地模型。

> **應用案例:** 想體驗「OpenClaw 式」的個人 AI 助理,卻被 43 萬行原始碼勸退——
> 用 NanoBot 在飛書或 Telegram 接一個 bot,直接用自然語言下達「幫我整理今天的待辦、提醒我下午三點開會」,
> 它靠記憶模組記住脈絡;若擔心 API 成本,把模型換成本地模型即可,整套程式碼僅約 4000 行,適合拿來讀懂與二次開發。

## 2. Beads — 給程式 Agent 的外部記憶與任務溯源工具

- **連結:** <https://github.com/stewyegge/beads>
- **用途:** 專為 **程式設計 Agent** 設計的外部記憶工具。用 AI 寫稍微複雜一點的專案時,很容易上下文超出、然後忘了前面在做什麼;傳統做法是用 Markdown 記「要做什麼、已做什麼」,而 Beads **用具備依賴感知的圖結構取代雜亂無章的 Markdown 任務清單**。
- **亮點:**
  - 提供 **顯式依賴管理、任務執行溯源、會話狀態持久化、多 Agent 任務隔離分配、語意化審計追蹤**。
  - 本質是一個 **基於 Git 的任務管理系統**:把任務拆得很細,存成 **JSON 格式** 放進你的倉庫,讓 AI 隨時知道「現在做到哪一步、下一步該做什麼」,**任務跨好幾天也不會斷片**。
  - 安裝走一鍵腳本或 npm,會 **全域安裝 `bd` 指令**(不放到具體專案)。
  - 用法:進到專案執行 `bd init` 自動初始化,遇到問題用 `bd doctor` 排查,最後把 `Use 'bd' for task tracking` 寫進 `AGENTS.md`,Agent 進入工具時就會自動用 `bd` 規劃與管理開發任務。

> **應用案例:** 用 Claude Code 或 Cursor 開發一個要做好幾天的功能,常常隔天 Agent 就「忘了」昨天的計畫——
> 在專案裡 `bd init` 後,把任務拆解、依賴關係與進度都存成 JSON 進 Git,
> Agent 重新進場時讀回任務圖,就能接著「下一個未完成且依賴已滿足」的任務繼續做,不用每次重貼背景。

## 3. ChatLab — 本地化的聊天記錄分析工具

- **連結:** <https://github.com/helledogua/ChatLab>
- **用途:** 一個 **免費、開源、本地化** 的聊天記錄分析應用。透過 **AI Agent + 靈活的 SQL 引擎**,讓你自由拆解、查詢甚至重構自己的社交數據。聊天記錄往往是個人資訊量最大的資料,放在本地分析就不會有隱私外洩疑慮。
- **亮點:**
  - 支援 **多種聊天軟體格式匯入**,後台轉成統一的標準化結構,資料存入 **本地 SQLite 資料庫**。
  - 內建 AI Agent,整合 **10+ 個 Function Calling 工具**,支援動態調度,深度挖掘聊天記錄。
  - 支援 **多維數據視覺化**:趨勢分析、統計、過濾,還能產出年度榜單等。
  - **不含資料匯出功能**:聊天記錄需自行用對應工具匯出後再匯入(文件中有說明支援的類型)。

> ⚠️ **使用聲明:** 分析聊天記錄涉及個人隱私與他人對話內容。請僅分析 **你有權處理的資料**,
> 並注意當地關於通訊隱私的法律與對方知情同意,切勿用於窺探或散布他人私訊。

> **應用案例:** 想知道自己和某位朋友這一年到底聊了多少訊息、誰先已讀不回、哪幾個月互動最頻繁——
> 把聊天記錄匯出後丟進 ChatLab,直接用自然語言問 AI Agent(它在背後轉成 SQL 查 SQLite),
> 就能拉出趨勢圖與年度榜單,全程在本地完成、不上傳雲端。

## 4. Vision-Agents — 面向即時影片/語音感知的 AI Agent 框架

- **連結:** <https://github.com/GetStream/Vision-Agents>
- **用途:** 由 **Stream(GetStream)** 開源的一套面向 **即時影片/語音感知** 的 AI Agent 框架。它不像普通視覺模型只輸出推理結果,而是提供一整套構建「可觀察影片、可聽音訊、可對話互動」智能體的模組與 SDK。
- **亮點:**
  - 重點在於讓開發者用 **自己選擇的模型** 快速搭建 **低延遲** 的視覺/語音智能體,不必為底層資料流處理與多模態協作重造輪子。
  - 把原本分散的音、視訊處理部分合理整合。
  - 開發時除了大模型 API key,還需準備 Stream 的 key(**有免費額度**)。
  - 官方附多個範例的對應原始碼可參考。

> **應用案例:** 想做一個「即時高爾夫教練」——攝影機拍下揮桿動作,Agent 即時辨識並用語音給出修正建議;
> 或做「快遞包裹防盜監測」:偵測人臉、追蹤包裹,在包裹被取走時即時發出警報。
> 用 Vision-Agents 搭配自選模型與 Stream 的串流基礎設施,就能快速把這類即時視覺/語音助理跑起來。

## 5. KeyStats — 鍵盤、滑鼠使用統計工具

- **連結:** <https://github.com/debugtheworldbot/keyStats>
- **用途:** 一款 **輕量級的 macOS / Windows 原生菜單列(menu bar)應用**。走路時我們用「微信運動」記步數,那坐在電腦前工作時呢?KeyStats 即時統計每日 **鍵盤敲擊次數、滑鼠點擊次數、滑鼠移動距離、滾動距離**,並把核心數據直接顯示在菜單列。
- **亮點:**
  - **資料完全本地**儲存。
  - 除了次數統計,還能看 **趨勢**、**按應用分類查看**(同一 App 內不再細分,例如全在瀏覽器工作就都歸瀏覽器)。
  - 提供 **有史以來的分析報告**。
  - 安裝:Mac 用 `brew`(先 `brew tap` 訂閱再 `brew install`);Windows 到 release 直接下載安裝包。

> **應用案例:** 想量化自己的「打字勞動量」或檢視重度滑鼠使用是否該換設備——
> 裝上 KeyStats 後,菜單列隨時看到當日敲了幾下鍵盤、滑鼠移動了多少距離;
> 玩射擊遊戲的人可能 WASD 特別多,玩 LOL/DOTA2 的人 QWER 特別多,長期下來還能看出自己的使用習慣趨勢。
